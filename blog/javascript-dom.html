JavaScript and its relationship to HTML and CSS... for humans

When creating websites, we're using three major components: HTML (a markup language), CSS (a styling language), and an actual programming language, JavaScript. One way to describe the relationship between this iconic trio is to compare it to the human body.

HTML is responsible for the overall structure and content of a website. It provides our "body" with all of the bones, working organs and limbs. For now, the body is alive but not yet moving or speaking.

CSS makes the websites pretty and positions elements according to the designer's preference. This is similar to choosing suitable clothes, a certain colour of nail polish and adding a pair of earrings.

Finally, JavaScript makes the page interactive and brigs them to life. In terms of the body analogy, JavaScript provides us with functionality â€“ at this point we can communicate and move.

What is DOM? Oh baby, don't hurt me! Don't hurt me... no more!

DOM or the Document Object Model is thoughtfully created by the browser when a website is opened. It creates a tree-like structure that JavaScript uses in order to communicate with HTML and try to interact with its elements.

DOM is just like a family tree with a few differences. Tragically, each element (or a node) only has one parent, but don't feel sorry for this family â€” children have numerous siblings and behave amazingly well and always do exactly as told by their parent! And let's just say that parents are very, very prolific. They can have as many children as they wish and looks like there's no birth control in the HTML kingdom.

Difference between accessing data from arrays and objects

An object is basically a thing. Any thing. This thing can have multiple characteristics: 

        
let student = {
    name: 'Elena',
    age: 35,
    enjoysWritingBlogs: false,
}

To get ahold of the data stored in the object we use the dot notation. 

        
 student.name //returns "Elena"

An array is a collection of elements. 

        
 let hobbies = ["hiking", "reading", "playing WoW"]

To get ahold of data stored in an array we can use the indexation. 

        
hobbies[0] // returns "hiking"

Functions. Why are we doing this to ourselves?

Functions are chunks of code that are created to perform a certain task. Whenever your code becomes repetitive, it might be a good idea to shove it into a brand new shiny function. What makes functions so hot? They are reusable and improve code readability (if you name them properly!).
