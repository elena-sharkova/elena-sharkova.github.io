CSS best practices:

classes vs IDs

How to create and use our selectors

Classes and IDs are the HTML attributes that we can assign to any of the HTML elements. The main difference between them is that ID is unique. Imagine a group of school kids having lunch. They are all bunched together in a "class" while everyone is, naturally, unique and their own person. Now, let's remind ourselves how to create our classes and IDs in HTML. 

            
<li class="test-class"></li>
<p id="test-id"></p>

And this is how we refer to our selectors in CSS. 

          
#test-id {
  text-align: center;
}

.test-class {
  font-size: 1rem;
}

In favour of classes

Currently, the general consensus is to prioritise using classes over IDs. There are several reasons behind this, with specificity being the number one. When applying CSS it is easiest to use the least specific selectors when possible. IDs being the most specific (with an exception of inline styling which is not considered to be a best practice) are harder to control. No amount of class styles will ever override an ID's styling.

Another reason is reusability. Yes, when you have just one particular button, it is very tempting to use an ID, but the chances are you might end up using the same button somewhere else. Classes (remember to apply good semantic naming) make styling or restyling elements much more convenient.

Moreover, you can stack as many classes as you want to, but can only use one ID per element. This is especially valuable when your CSS gets more complicated and when you start using CSS libraries. Note that elements can have both, an ID and class/classes.

In favour of IDs

All of the above does not mean that IDs have no place in our code. IDs bring something really unique (pun intended) to the table â€” they can interact with a browser. When a section has an assigned ID, we can use it as an anchor in order to "jump" right there scrolling past all of the content. An example of a website link using this technique would be:

https://yoursite.com.index.html#contact

IDs are also used by JavaScript when you need that access and manipulate that specific element on your page and not their whole class.

And last but not least, one of the examples of using IDs for increased accessibility is to connect labels to their respective form fields. Check out Web Accessibility Initiative's instructions for further reading.

Conclusion

While the convention is to try and reuse classes, IDs still provide some unique features.
