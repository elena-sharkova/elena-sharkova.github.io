CSS best practices:

classes vs IDs

How to create and use our selectors

Classes and IDs are the HTML attributes that we can assign to any of the HTML elements. The main difference between them is that ID is unique. Imagine a group of school kids having lunch in a group. They are all bunched together in a "class" while everyone is, naturally, unique and their own person. Now, let's remind ourselves how to create our classes and IDs in HTML.

            
<li class="test-class"></li>
<p id="test-id"></p>

And this is how we refer to our selectors in CSS. 

          
#test-id {
  text-align: center;
}

.test-class {
  font-size: 1rem;
}

In favour of classes

Currently, the general consensus is to prioritise using classes over IDs. There are several reasons behind this, with specificity being the number one. When applying CSS it's easiest to use the least specific selectors when possible. IDs being the most specific (with an exception of inline styling which is not considered to be a best practice) are harder to control. No amount of class styles will ever override an ID's styling.
